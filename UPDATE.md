# 업데이트 배포 & 버전 안내 가이드

앱은 시작할 때 **Firestore `config/app` 문서의 `latest`** 를 읽어 자기 버전과 비교하고,
더 높으면 **업데이트 배너**("다시 받아주세요 + 다운로드 버튼")를 띄웁니다. (`force:true`면 강제)

> ⚠️ 업데이트 체커는 **1.0.1부터** 들어갑니다. 이미 배포된 1.0.0 사용자는 이 알림을 못 받아요(체커가 없으니). 1.0.1을 한 번 배포한 뒤부터 자동 안내가 작동합니다.

---

## A. 최초 1회 — Firebase 콘솔 설정

### 1) 보안 규칙 다시 게시
Firestore → 규칙 탭에 `firestore.rules` 내용을 붙여넣고 **게시** (config 읽기 허용 규칙이 추가됨).

### 2) 설정 문서 만들기
Firestore → 데이터 → **컬렉션 시작** `config` → **문서 ID** `app` → 필드 4개:

| 필드 | 타입 | 값(예) |
|------|------|--------|
| `latest` | string | `1.0.1` |
| `url` | string | `https://woopo666.github.io/bird-pocket/` |
| `message` | string | `새 버전이 나왔어요! 다시 받아주세요 🐦` |
| `force` | boolean | `false` |

→ 저장. (지금은 `latest`=현재 버전이라 아무 배너도 안 뜸 = 정상)

---

## B. 새 버전 낼 때마다 (예: 1.0.1 → 1.0.2)

1. 코드 수정
2. `package.json` 의 `"version"` 을 올림 (예: `1.0.2`)
3. 빌드: `npm run dist` (Windows .exe) / 맥에서 `npm run dist:mac` (.dmg)
4. **GitHub 릴리스**에 새 파일 업로드 (새 태그 `v1.0.2`)
5. 다운로드 페이지(`docs/index.html`)의 다운로드 링크를 새 파일명으로 수정 → 커밋
6. **Firestore `config/app` 의 `latest` 를 `1.0.2` 로 변경** (필요하면 `message`도 수정)

→ 6번 즉시, 기존 사용자(1.0.1 이하)가 앱을 켜면 **"다시 받아주세요" 배너**가 뜨고,
   버튼을 누르면 다운로드 페이지가 열립니다. (`force:true`로 두면 플레이 전 강제 안내)

> 클라우드 저장이라, 사용자가 새 버전을 받아도 **진행도는 계정에 그대로** 유지됩니다.

---

## 동작 요약
- `latest` > 설치된 버전  → 상단 배너 (다운로드 / 나중에)
- `force: true`          → 전체 화면 강제 안내 (다운로드만)
- `latest` ≤ 설치된 버전  → 아무것도 안 뜸
